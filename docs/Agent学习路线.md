# 前端工程师从 0 到 1 搭建 Agent 学习路线

> 面向有前端（JS/TS）背景的开发者，从理解原理到亲手搭建一个完整可用的 Agent。

## 前端的天然优势

- 熟悉 JS/TS 生态
- 对异步、流式数据（SSE）敏感
- 能快速做出可视化界面，调试和演示都方便

---

## 一、先搞懂「Agent 到底是什么」

别急着写代码，先建立心智模型。一个 Agent 的本质就是一个循环：

```
while (任务未完成) {
  1. 把「上下文 + 可用工具」发给 LLM
  2. LLM 返回：要么直接回答，要么决定调用某个工具
  3. 如果调工具 → 执行工具 → 把结果塞回上下文
  4. 回到第 1 步
}
```

核心三件套：**LLM（大脑）+ Tools（手脚）+ Loop（循环调度）**。
理解这个循环，比看任何框架都重要。

## 二、分阶段学习路线

### 阶段 0：打基础（1-2 天）

- 注册一个能用的 LLM API（OpenAI / DeepSeek / 通义 / Claude 等都行，国内推荐 DeepSeek，便宜）
- 用 `fetch` 直接调一次 chat completions 接口，跑通「发消息 → 拿回复」
- 重点理解：`messages` 数组结构、`role`（system / user / assistant / tool）、流式返回（SSE）

### 阶段 1：手写一个最小 Agent（核心，必做，3-5 天）

**不要用任何框架**，纯手写。这是真正学到东西的阶段。依次实现：

1. **单轮对话** → 多轮对话（维护 messages 历史）
2. **Function Calling**：定义一个工具（如「查天气」「读文件」），让 LLM 决定何时调用
3. **Agent Loop**：把工具调用结果塞回去，让模型继续，直到任务完成
4. **加 2-3 个真实工具**：比如「执行 shell 命令」「读写本地文件」「网络搜索」

做完这个，你就拥有了一个能干活的 mini coding agent。

### 阶段 2：工程化增强（1 周）

- **System Prompt 设计**：角色、规则、工具使用约束
- **上下文管理**：历史太长怎么压缩 / 截断（滑动窗口、摘要）
- **流式输出**：边生成边显示（前端优势，做个聊天 UI）
- **错误处理与重试**：工具报错、模型幻觉调用不存在的工具
- **ReAct 模式**：让模型显式输出「思考过程」再行动

### 阶段 3：进阶概念（按需）

- **MCP（Model Context Protocol）**：现在的工具标准化协议，很值得学
- **RAG**：接入向量库做知识检索（embedding + 相似度搜索）
- **多 Agent 协作**、记忆持久化、规划（Planning）

## 三、技术选型（贴合前端背景）

| 用途 | 推荐 |
| --- | --- |
| 语言 | **TypeScript**（主场） |
| 运行时 | Node.js（先 CLI），后接前端 UI |
| 阶段 1 | 裸 `fetch` / `openai` SDK，**不用框架** |
| 阶段 2+ 框架 | **Vercel AI SDK**（前端友好，强烈推荐）或 LangChain.js |
| UI | React / Vue 都行，做个对话界面 |

---

## 四、具体建议

1. **最关键的一条：阶段 1 一定要纯手写，别一上来用 LangChain。**
   框架会把核心循环藏起来，你学不到本质。手写 100 行代码胜过看 10 篇教程。

2. **以「能跑的小项目」驱动学习。**
   建议第一个目标项目：做一个**命令行版的文件助手 Agent**——能读本地代码、回答关于代码的问题、按要求改文件。麻雀虽小五脏俱全。

3. **善用正在用的工具。**
   你现在用的 IDE 助手本身就是个成熟 Agent，可以边用边观察它怎么调用工具、怎么读文件、怎么循环。

4. **推荐两个入门读物：**
   - Anthropic 的《Building Effective Agents》（讲清了 workflow vs agent）
   - Vercel AI SDK 官方文档的 Agents 章节（带 TS 代码）

---

## 五、第一个动手项目目标

> 命令行版「文件助手 Agent」

具备能力：
- 读取本地代码文件
- 回答关于代码的问题
- 按要求修改文件

技术组合：TypeScript + Node.js + 裸 `fetch` + 一个 Agent Loop + 2 个工具（读文件、写文件）。
